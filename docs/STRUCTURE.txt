📦 RACINE DU PROJET
├── 📄 index.html                    # Landing page (page d'accueil)
├── 📄 login.html                    # Authentification (Google + Email/Password)
├── 📄 onboarding.html               # Configuration profil après 1ère connexion
├── 📄 chat.html                     # App principale (charge les modules)
├── 📄 _redirects                    # Routing Netlify
├── 📄 netlify.toml                  # Config Netlify
├── 📄 README.md                     # Documentation déploiement
│
├── 📁 assets/                       # Ressources statiques
│   ├── 📁 images/
│   │   ├── eva.svg                  # Logo Eva (animé)
│   │   ├── logo.svg                 # Logo Astral Technology
│   │   └── favicon.svg              # Icône navigateur
│   │
│   ├── 📁 styles/
│   │   ├── main.css                 # Styles globaux (variables, reset)
│   │   ├── sidebar.css              # Styles sidebar Grok-like
│   │   ├── chat.css                 # Zone messages et input
│   │   ├── modals.css               # Modals (paramètres, rapports, etc.)
│   │   └── responsive.css           # Media queries mobile/tablet
│   │
│   └── 📁 sounds/
│       ├── notification.mp3         # Son notification
│       └── alarm.mp3                # Son alarme par défaut
│
├── 📁 js/                           # Scripts JavaScript modulaires
│   ├── 📁 core/
│   │   ├── firebase-config.js       # Config Firebase + init
│   │   ├── auth.js                  # Gestion authentification
│   │   ├── config.js                # Configuration utilisateur (DEFAULT_CFG)
│   │   └── utils.js                 # Fonctions utilitaires (toast, formatDate, etc.)
│   │
│   ├── 📁 ai/
│   │   ├── providers.js             # Gestion providers IA (Qwen, Puter, OpenAI, etc.)
│   │   ├── chat-handler.js          # Envoi messages + réception réponses
│   │   └── vision.js                # Analyse d'images (Moondream, GPT-4o)
│   │
│   ├── 📁 voice/
│   │   ├── tts.js                   # Text-to-Speech (Microsoft Denise, Puter, etc.)
│   │   ├── stt.js                   # Speech-to-Text (Web Speech API)
│   │   └── wake-word.js             # Détection "Hey Eva"
│   │
│   ├── 📁 ui/
│   │   ├── sidebar.js               # Gestion sidebar (toggle, navigation)
│   │   ├── chat-ui.js               # Affichage messages, scroll auto
│   │   ├── modals.js                # Ouverture/fermeture modals
│   │   └── animations.js            # Animations logo Eva
│   │
│   ├── 📁 features/
│   │   ├── conversations.js         # Gestion conversations Firestore
│   │   ├── alarms.js                # Système alarmes
│   │   ├── reminders.js             # Système rappels
│   │   ├── calendar.js              # Agenda/événements
│   │   ├── notes.js                 # Prises de notes
│   │   └── reports.js               # Système de rapports bugs/feedback
│   │
│   ├── 📁 settings/
│   │   ├── settings-ui.js           # Interface paramètres (modal Grok-like)
│   │   ├── profile.js               # Gestion profil utilisateur
│   │   └── dev-keys.js              # Activation clés développeur
│   │
│   └── app.js                       # Point d'entrée principal (init app)
│

    ├── DEPLOIEMENT.md               # Guide déploiement Netlify
    ├── FIREBASE_SETUP.md            # Configuration Firebase
    ├── DEV_KEYS_GUIDE.md            # Guide clés développeur
    └── TROUBLESHOOTING.md           # Résolution problèmes courants


═══════════════════════════════════════════════════════════
🔥 FICHIERS PRINCIPAUX (HTML)
═══════════════════════════════════════════════════════════

📄 index.html
  → Landing page moderne
  → Header avec logo + CTA "Se connecter"
  → Section hero avec présentation Eva
  → Footer avec infos Astral Technology

📄 login.html
  → Authentification Google + Email/Password
  → "Rester connecté" (persistence)
  → Redirect vers onboarding.html si nouveau compte
  → Redirect vers chat.html si compte existant

📄 onboarding.html
  → Configuration profil (nom, surnom, avatar)
  → Choix préférences initiales (voix, langue)
  → Redirect vers chat.html après configuration

📄 chat.html
  → Structure HTML minimale
  → Charge TOUS les modules JS (via <script type="module">)
  → Sidebar + Zone chat + Modals
  → 100% responsive mobile


═══════════════════════════════════════════════════════════
🎨 STYLES CSS (MODULAIRES)
═══════════════════════════════════════════════════════════

📄 main.css
  → Variables CSS (couleurs, espacements)
  → Reset CSS
  → Typographie (fonts Orbitron + Space Mono)
  → Styles globaux (body, animations de base)

📄 sidebar.css
  → Sidebar Grok-like
  → Sections (Chat, Voix, Imagine, Agenda, etc.)
  → Liste conversations
  → Profil utilisateur en bas

📄 chat.css
  → Zone messages (bulles user/assistant)
  → Input bar (textarea + boutons)
  → Logo Eva animé en haut

📄 modals.css
  → Modal paramètres (fullscreen, sections à gauche)
  → Modal rapports (formulaire)
  → Modal alarmes/rappels/notes/agenda

📄 responsive.css
  → Mobile (<768px) : Sidebar overlay + hamburger
  → Tablet (768-1024px) : Sidebar rétractable
  → Desktop (>1024px) : Sidebar fixe

═══════════════════════════════════════════════════════════
📱 RESPONSIVE MOBILE
═══════════════════════════════════════════════════════════

Mobile (<768px)
  ├─ Sidebar en overlay (slide depuis la gauche)
  ├─ Hamburger menu en haut à gauche
  ├─ Logo Eva centré (taille réduite)
  ├─ Input bar sticky en bas (safe-area pour notch)
  └─ Boutons plus gros (44px minimum)

Tablet (768-1024px)
  ├─ Sidebar rétractable (60px collapsed)
  ├─ Logo Eva visible
  └─ Boutons normaux (38px)

Desktop (>1024px)
  ├─ Sidebar fixe 260px
  ├─ Logo Eva pleine taille
  └─ Boutons normaux (36px)

🎯 AVANTAGES DE CETTE STRUCTURE
═══════════════════════════════════════════════════════════

✅ Maintenabilité
   → Chaque fichier a UNE responsabilité
   → Facile de trouver et modifier du code

✅ Performance
   → Chargement modulaire (import uniquement ce qui est nécessaire)
   → Mise en cache optimisée (fichiers statiques séparés)

✅ Collaboration
   → Plusieurs devs peuvent travailler en parallèle
   → Moins de conflits Git

✅ Debugging
   → Plus facile d'isoler les bugs
   → Console indique le fichier exact de l'erreur

✅ Scalabilité
   → Facile d'ajouter de nouvelles fonctionnalités
   → Juste créer un nouveau fichier dans le bon dossier
